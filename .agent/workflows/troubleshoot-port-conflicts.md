---
description: ポート競合（EADDRINUSE）の解消手順
---

開発中に「サーバーが起動しない」「古いコードが反映されない」といった問題が発生した際、古いプロセスがポートを掴んだままになっている（ゾンビプロセス）可能性があります。以下の手順で解消できます。

### 1. 使用中のポートを確認する
ターミナルで以下のコマンドを実行し、どのプロセスがポートを使用しているか確認します。
（本プロジェクトでは主に 3001, 5173 を使用します）

```bash
lsof -i :3001,5173
```

### 2. プロセスを強制終了する
もしプロセスが残っている場合は、以下のコマンドで一括して強制終了できます。

```bash
# ポート 3001 と 5173 を使用しているプロセスをすべて終了
lsof -t -i :3001,5173 | xargs kill -9
```

### 3. サーバーを再起動する
すべてのポートが空いたことを確認してから、再度起動します。

```bash
pnpm dev
```

> [!TIP]
> もし Vite が 5174 や 5175 で起動してしまった場合は、必ず一度 `kill` コマンドを実行して 5173 を空けてから再起動することをお勧めします。
