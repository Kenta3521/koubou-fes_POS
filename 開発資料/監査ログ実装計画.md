# 監査ログ実装計画書

**作成日:** 2026年2月8日  
**ステータス:** 草案（承認待ち）

---

## 1. 目的
監査ログ仕様書に基づき、システムの透明性と安全性を確保するため、重要操作の履歴を記録し、適切に表示する機能を実装する。

---

## 2. データベース変更 (Schema)

### 2.1 AuditLog テーブルの拡張
[schema.prisma](file:///Users/iwatasouya/koubou-fes_POS_root/apps/api/prisma/schema.prisma) に以下のフィールドを追加します。

| フィールド | 型 | デフォルト | 説明 |
| :--- | :--- | :--- | :--- |
| `isSystemAdminAction` | Boolean | `false` | システム管理者による特権操作かどうかのフラグ |
| `category` | String | (なし) | 操作カテゴリ (例: `PRODUCT`, `DISCOUNT`) |

---

## 3. バックエンド実装 (API)

### 3.1 監査サービス (`AuditService`) の新規作成
共通ロジックを [auditService.ts](file:///Users/iwatasouya/koubou-fes_POS_root/apps/api/src/services/auditService.ts) に集約します。
- **トランザクション対応:** `prisma.$transaction` 内で実行できるよう、`Prisma.TransactionClient` を受け取れる設計にする。
- 操作実行者の権限を確認し、自動的に `isSystemAdminAction` を設定。

### 3.2 既存機能へのログ埋め込み
以下の各サービス/コントローラーの主要アクション完了後に `AuditService` を呼び出します。
**原則として同一トランザクション内で記録します。**

- **認証:** ログイン、パスワード変更、パスワードリセット、プロフィール更新。
- **団体管理:** 団体作成、更新、招待コード再発行。
- **メンバー管理:** 団体加入、脱退、追放、権限変更。
- **商品・カテゴリ:** 商品作成、編集、削除。カテゴリ作成、編集、削除、並び替え。
- **割引管理:** 割引設定の作成、編集、削除。
- **取引:** キャンセル、返金処理（PayPay）。
- **レジ締め:** レジ締め報告の作成。
- **RBAC:** ロール/権限の定義変更、ロール詳細変更、ロール割り当て。

### 3.3 監査ログ取得 API の実装
[auditController.ts](file:///Users/iwatasouya/koubou-fes_POS_root/apps/api/src/controllers/auditController.ts) を新規作成します。
- `GET /api/v1/organizations/:orgId/audit-logs`: 団体管理者用（CSVエクスポート対応）。
- `GET /api/v1/admin/audit-logs`: システム管理者用（CSVエクスポート対応）。

## 4. フロントエンド実装 (Web)

### 4.1 フォーマッタとコンポーネント
- **`AuditLogFormatter`:** JSON形式のペイロードを日本語の文章に変換。
- **`AuditLogTable`:** 検索・フィルタリング機能を備えたログ一覧テーブル。
    - **CSVチャット**: 表示データをCSVでダウンロードするボタンを実装。
- **`AuditLogDetailModal`**: ログの詳細を表示するモーダル。
    - **`DiffViewer`**: `changes` オブジェクトを解析し、変更前後の値を比較表示するコンポーネント。

### 4.2 既存画面への「理由入力」実装
以下の操作を行う際、ユーザーに「操作理由」を入力させるUI（モーダル等）を追加改修します。
- **取引キャンセル**: `PayPayPaymentPage` 等のキャンセル確認ダイアログに理由入力欄を追加。
- **返金処理**: 返金実行時の確認モーダルに理由入力欄を追加。
- **メンバー強制脱退**: 管理画面の脱退処理に理由入力欄を追加。
- **メンバー自発脱退**: 脱退確認時にアンケート的な理由入力欄を追加（任意）。

### 4.3 管理画面への統合
1.  **団体管理画面:** 
    - 独立した「監査ログ」ページ。
2.  **システム管理者ポータル:**
    - 全体監査ログフィード。
    - 団体詳細画面内の「監査ログ」タブ。
    - システム管理者自身の操作履歴確認ビュー。
