# 割引および合計金額の計算仕様

本ドキュメントでは、POSシステムにおける注文合計金額および割引の計算ロジック（`calculationService.ts`）について解説します。

## 1. 計算の全体フロー

計算は以下のステップで実行されます。

1.  **バリデーション**: 商品の存在、所属組織、有効状態（isActive）、論理削除状態、および在庫を確認します。
2.  **有効な割引の抽出**: 現在時刻において有効（開始前・終了後でない）かつ、有効フラグがONの割引を取得します。
3.  **カテゴリ単位の集計**: カート内の商品をカテゴリごとに集計（合計数量・合計金額）します。これは「カテゴリ単位の判定条件」に使用されます。
4.  **商品ごとの計算（個別割引）**: 各商品に対して、適用可能な「自動割引」と「指定された手動割引」を評価し、最も優先度の高いものを適用します。
5.  **小計の計算**: 個別割引適用後の商品金額を合計します。
6.  **全体割引の適用**: カート全体（ORDER_TOTAL）を対象とした手動割引を評価し、最終的な合計金額を算出します。

---

## 2. 割引の適用ロジック（分離された条件と対象）

本システムでは、割引の「判定条件」と「適用対象」が独立して定義されています。

### 判定条件（トリガー）の評価
各割引には、以下のいずれかの判定条件が設定されます。

- **注文合計 (ORDER_TOTAL)**: カート全体の金額や数量で判定。
- **特定カテゴリ (CATEGORY)**: 特定のカテゴリに属する商品の合計数量や合計金額で判定。
- **特定商品 (SPECIFIC_PROD)**: 特定の商品がカートに含まれているか、その数量で判定。

これらに対して、「なし（常に適用）」「最低数量」「最低金額」のしきい値チェックが行われます。

### 適用対象（アクション）の評価
判定条件が満たされた場合、その割引がどの項目に適用されるかを判定します。

- **特定商品**: 指定された商品IDと一致する場合に適用。
- **カテゴリ全体**: 指定されたカテゴリIDに商品が属する場合に適用。
- **注文合計**: 全ての個別割引計算が終わった後の、注文最終金額に適用。

---

## 3. 割引の優先度と競合

1つの商品に対して複数の割引（自動割引、およびその商品が対象の手動割引）が適用可能な場合、以下のルールで適用される1つが決定されます。

1.  **優先度（Priority）**: `priority` フィールドの数値が高い割引が優先されます。
2.  **適用数**: 商品1点につき、適用される割引は最大で**1つ**です。（重複して適用されることはありません）

---

## 4. 計算項目の定義

最終的に返却されるデータには以下の項目が含まれます。

| 項目名 | 説明 | 計算式 |
| :--- | :--- | :--- |
| **subtotalAmount** | 割引前の定価合計 | Σ(商品の定価 × 数量) |
| **totalDiscountAmount** | 割引額の総計 | 個別割引の合計 + 全体割引 |
| **totalAmount** | 最終的な支払い合計金額 | subtotalAmount - totalDiscountAmount |

### 金額計算の端数処理
- 百分率（%）割引の場合、割引額の算出には `Math.floor` を使用し、円単位で切り捨てます。
- マイナス価格防止ロジックにより、合計金額が0円を下回ることはありません。
