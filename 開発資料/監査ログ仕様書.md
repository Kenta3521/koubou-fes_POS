# 監査ログ仕様書

**作成日:** 2026年2月8日  
**最終更新:** 2026年2月16日  
**ステータス:** レビュー中（v2）  
**目的:** 不正防止、トラブル時の原因究明、および運営の透明性確保のため、システム内で行われる重要な操作を記録・表示する仕様を定義する。

---

## 1. 監査対象ユーザーと閲覧権限

監査ログは「誰が・いつ・何をしたか」を記録し、適切な権限を持つユーザーが確認できるようにする。

| ロール | 閲覧範囲 | 主な用途 |
| :--- | :--- | :--- |
| **システム管理者** (実行委員会) | **全体** (全団体 + システム全般) | 不正の監視、システム設定変更の追跡、全団体の運営状況確認 |
| **団体管理者** (各店責任者) | **自団体のみ** | スタッフの操作ミス、不正な値引き/在庫調整、返金処理の確認 |

---

## 2. 記録データ形式 (Data Schema)

### 2.1 AuditLog テーブル

| フィールド名 | 型 | 制約 | 説明 |
| :--- | :--- | :--- | :--- |
| `id` | UUID | PK | ログID |
| `userId` | UUID | FK (User) | 操作実行者 |
| `organizationId` | UUID? | FK (Organization) | 関連団体ID。システム全体の操作はNULL。 |
| `category` | String | NOT NULL | 操作カテゴリ。フィルタリング高速化用。 |
| `action` | String | NOT NULL | 定義済みの `AuditAction` コード（セクション3参照） |
| `targetId` | String? | | 操作対象のID（商品ID、ユーザーID、取引ID等） |
| `payload` | JSON? | | 詳細情報。変更前後の値、IPアドレス、ユーザーエージェント等。 |
| `isSystemAdminAction` | Boolean | DEFAULT: false | システム管理者による操作の場合にtrue。 |
| `createdAt` | DateTime | DEFAULT: now() | 記録日時 |

### 2.2 Prisma スキーマ

```prisma
model AuditLog {
  id                  String        @id @default(uuid())
  organizationId      String?
  userId              String
  category            String
  action              String
  targetId            String?
  payload             Json?
  isSystemAdminAction Boolean       @default(false)
  createdAt           DateTime      @default(now())
  organization        Organization? @relation(fields: [organizationId], references: [id])
  user                User          @relation(fields: [userId], references: [id])

  @@index([organizationId, createdAt])
  @@index([category])
  @@index([userId])
}
```

### 2.3 カテゴリ一覧

TypeScript型として定義（DB上はString）：

```typescript
type AuditCategory = 'AUTH' | 'USER' | 'PRODUCT' | 'DISCOUNT' | 'ROLE' | 'TRANSACTION' | 'ORG' | 'SYSTEM';
```

> [!IMPORTANT]
> **isSystemAdminAction の重要性**  
> 実行委員会（システム管理者）による操作を明示的にマークすることで、特定団体への不適切な介入や職権濫用を監視できるようにする。

## 3. 監査対象アクション一覧

### 3.1 認証・ユーザー管理 (Category: AUTH / USER)
| アクションコード | 説明 | 記録内容 (payload) |
| :--- | :--- | :--- |
| `USER_REGISTER` | ユーザー新規登録 | `{ "email": "...", "name": "..." }` |
| `LOGIN_SUCCESS` | ログイン成功 | `{ "ip": "...", "userAgent": "..." }` |
| `LOGIN_FAILURE` | ログイン失敗 | `{ "ip": "...", "email": "..." }` |
| `USER_PROFILE_UPDATE` | プロフィール更新 | `{ "changes": { "email": { "old": "...", "new": "..." } } }` |
| `PASSWORD_CHANGE` | パスワード変更 | `{ "ip": "..." }` |
| `MEMBER_JOIN` | 団体への新規加入(招待) | `{ "inviteCode": "...", "newMemberId": "..." }` |
| `MEMBER_ADD` | メンバー追加(管理者) | `{ "newMemberId": "...", "addedBy": "..." }` |
| `MEMBER_UPDATE` | メンバー情報更新 | `{ "targetUserId": "...", "changes": { "role": "...", "permissions": "..." } }` |
| `MEMBER_REMOVE` | メンバーの強制脱退 | `{ "targetUserId": "...", "reason": "..." }` |
| `MEMBER_LEAVE` | 団体からの脱退 | `{ "reason": "..." }` |

### 3.2 商品・在庫・カテゴリ (Category: PRODUCT)
| アクションコード | 説明 | 記録内容 (payload) |
| :--- | :--- | :--- |
| `CATEGORY_CREATE` | カテゴリ作成 | `{ "name": "..." }` |
| `CATEGORY_UPDATE` | カテゴリ更新 | `{ "categoryId": "...", "changes": { "name": "..." } }` |
| `CATEGORY_DELETE` | カテゴリ削除 | `{ "categoryId": "...", "name": "..." }` |
| `CATEGORY_REORDER` | カテゴリ並び替え | `{ "order": ["id1", "id2"] }` |
| `PRODUCT_CREATE` | 商品の新規作成 | `{ "name": "...", "price": 100, "categoryId": "..." }` |
| `PRODUCT_UPDATE` | 商品情報の変更 | `{ "productId": "...", "changes": { "price": { "old": 100, "new": 120 } } }` |
| `PRODUCT_DELETE` | 商品の削除 | `{ "productId": "...", "name": "...", "deletedAt": "..." }` |

### 3.3 割引設定 (Category: DISCOUNT)
| アクションコード | 説明 | 記録内容 (payload) |
| :--- | :--- | :--- |
| `DISCOUNT_CREATE` | 割引の作成 | `{ "name": "...", "value": 100, "type": "FIXED" }` |
| `DISCOUNT_UPDATE` | 割引の変更 | `{ "discountId": "...", "changes": { "value": { "old": 100, "new": 200 } } }` |
| `DISCOUNT_DELETE` | 割引の削除 | `{ "discountId": "...", "name": "..." }` |

### 3.4 権限・ロール管理 (Category: ROLE)
| アクションコード | 説明 | 記録内容 (payload) |
| :--- | :--- | :--- |
| `ROLE_CREATE` | ロールの作成 | `{ "roleName": "...", "permissions": [...] }` |
| `ROLE_UPDATE` | ロールの変更 | `{ "roleId": "...", "changes": { "name": { "old": "...", "new": "..." } } }` |
| `ROLE_DELETE` | ロールの削除 | `{ "roleId": "...", "roleName": "..." }` |
| `ROLE_PERM_UPDATE` | ロール権限の変更 | `{ "roleId": "...", "added": [...], "removed": [...] }` |

### 3.5 取引・会計 (Category: TRANSACTION)

> [!NOTE]
> 取引の作成・完了は `Transaction` テーブルの履歴として記録されるため、監査ログの対象外とする。
> 監査ログではキャンセル・返金・レジ締めなど、**運営上の判断が伴う操作**のみを記録する。

| アクションコード | 説明 | 記録内容 (payload) |
| :--- | :--- | :--- |
| `TRANSACTION_CANCEL` | 取引キャンセル | `{ "transactionId": "...", "reason": "..." }` |
| `TRANSACTION_REFUND` | 返金処理 | `{ "transactionId": "...", "amount": 1000, "reason": "..." }` |
| `CASH_REPORT_CREATE` | レジ締め報告 | `{ "theoretical": 10000, "actual": 10000, "diff": 0 }` |

### 3.6 団体・システム管理 (Category: ORG / SYSTEM)
| アクションコード | 説明 | 記録内容 (payload) |
| :--- | :--- | :--- |
| `ORG_CREATE` | 団体作成 | `{ "name": "...", "inviteCode": "..." }` |
| `ORG_UPDATE` | 団体情報更新 | `{ "changes": { "name": "...", "isActive": "..." } }` |
| `ORG_INVITE_REGEN` | 招待コード再発行 | `{ "oldCode": "...", "newCode": "..." }` |

## 4. API エンドポイント

### 4.1 団体管理者用

```
GET  /api/v1/organizations/:orgId/audit-logs
GET  /api/v1/organizations/:orgId/audit-logs/export
```

**共通 Query Parameters:**

| パラメータ | 型 | 説明 |
| :--- | :--- | :--- |
| `startDate` | string (ISO 8601) | 検索開始日 |
| `endDate` | string (ISO 8601) | 検索終了日（その日の23:59:59まで含む） |
| `category` | string | カテゴリフィルタ（AUTH, PRODUCT等） |
| `userId` | string | 操作者フィルタ |
| `page` | number | ページ番号（一覧のみ、デフォルト: 1） |
| `limit` | number | 件数（一覧のみ、デフォルト: 20） |

**一覧 Response:**
```json
{
  "success": true,
  "data": [
    {
      "id": "uuid",
      "userId": "uuid",
      "organizationId": "uuid",
      "category": "PRODUCT",
      "action": "PRODUCT_CREATE",
      "targetId": "uuid",
      "payload": { "name": "たこ焼き", "price": 300 },
      "isSystemAdminAction": false,
      "createdAt": "2026-02-16T00:00:00Z",
      "user": { "name": "田中太郎", "email": "tanaka@example.com" }
    }
  ],
  "meta": { "total": 100, "page": 1, "limit": 20, "totalPages": 5 }
}
```

**エクスポート Response:** CSV (UTF-8 BOM付き)

### 4.2 システム管理者用

```
GET  /api/v1/admin/audit-logs
GET  /api/v1/admin/audit-logs/export
```

**追加 Query Parameters（上記共通に加えて）:**

| パラメータ | 型 | 説明 |
| :--- | :--- | :--- |
| `organizationId` | string | 団体フィルタ |

**Response:** 団体管理者用と同構造。`organization` フィールドが追加。

## 5. ログの表示・検索要件

### 5.1 共通機能
*   **CSVエクスポート機能:**
    *   表示中のフィルタ条件に基づき、監査ログをCSV形式でダウンロード可能にする。
    *   CSVカラム: ID, 日時, カテゴリ, 操作, 実行者, 実行者Email, 対象ID, 詳細(JSON)
    *   UTF-8 BOM付き（Excel文字化け対策）。

### 5.2 団体管理者向け画面
*   自団体のログのみ表示。
*   カテゴリ、操作者、期間でのフィルタリング。
*   Payloadの日本語化表示（Formatter実装）。

### 5.3 UIデザイン (Display Guidelines)
*   **一覧画面 (Table View):**
    *   **カラム**: 日時, カテゴリ, 操作種別, 操作者, 概要(フォーマット済みメッセージ)
    *   **操作**: 「詳細」ボタンを各行に配置。
*   **詳細モーダル (Detail Modal):**
    *   **メタデータ**: IPアドレス, UserAgent, 対象IDなどを表示。
    *   **変更履歴 (Diff Viewer):**
        *   `changes` フィールドがある場合、変更前(`old`)と変更後(`new`)を左右または上下に並べて比較表示する。
        *   追加(`new`のみ)や削除(`old`のみ)の場合も視覚的に分かりやすく表示（緑/赤のハイライト等）。

## 6. 実装に関する技術指針

### 6.1 トランザクション内記録 (Strict Consistency)
*   監査ログの書き込みは、対象となる操作（商品作成など）と**同一のトランザクション**内で実行する。
*   **ログ書き込みが失敗した場合、メインの操作もロールバック（失敗）させる**。これにより、「操作は成功したがログが残っていない」という不整合を防ぐ。
*   エラーログはサーバーのシステムログ（標準出力/Datadog等）に出力し、管理者が検知できるようにする。

### 6.2 自動記録
*   Prisma Middleware または Service層のラッパー関数を用いて、実装漏れを防ぐ。

### 6.3 パフォーマンス
*   以下のインデックスを設定し、フィルタリング・ソートの高速化を図る：
    *   `@@index([organizationId, createdAt])` — 団体ごとの時系列クエリ
    *   `@@index([category])` — カテゴリフィルタ
    *   `@@index([userId])` — 操作者フィルタ

## 7. セキュリティ・保存ポリシー

### 7.1 保存期間
*   **無期限保存**とする。文化祭POSシステムは短期イベント用のため、データ量はイベント期間分のみで管理可能。

### 7.2 改ざん防止
*   監査ログテーブルに対する `DELETE` / `UPDATE` 操作は、アプリケーション層で禁止する。
*   監査ログのAPIに削除・更新エンドポイントは**設けない**。
*   DBレベルでも書き込み専用の権限設定を推奨。
